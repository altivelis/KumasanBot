# 要件定義書 — KumasanBot

## 目的・背景

Discordサーバー上で、VCへの参加を促進するためのポイント経済システムを構築する。
VCに滞在するとポイント（P）が貯まり、ガチャ・ギャンブル・送金・ショップなど多彩な機能でポイントを使い回せる仕組みを提供する。

---

## 技術スタック

| 項目 | 内容 |
|------|------|
| 言語 | JavaScript (Node.js) |
| フレームワーク | discord.js v14 |
| 設定管理 | `.env`（dotenv） |
| データストア | SQLite（better-sqlite3） |

---

## 機能要件

### 1. VC滞在ポイントシステム

- VCに参加している間、**1分ごとに1P**を付与する
- ポイントの集計はBot内部でタイマー管理する
- **計測対象**: Botアカウント（読み上げBotや音楽Botなど）は計測対象外とし、人間ユーザーのみを対象とする
- **カウント対象外チャンネル**: 管理者がカテゴリ単位で除外設定できる（「別募カテゴリ」「寝落ちVCカテゴリ」など）
- ユーザーがVC入退室したタイミングで計測開始・終了する

### 2. 送金・請求システム

- `/pay @ユーザー <金額>` : 指定ユーザーへPを送金する
- `/request @ユーザー <金額>` : 指定ユーザーへPを請求する（承認/拒否ボタン付き）
- 残高不足の場合はエラーを返す

### 3. ガチャシステム

- ガチャ券は**1時間VC滞在ごとに1枚**発行される
- **統合パネル方式**: 管理者が `/admin gacha panel` コマンドで指定チャンネルにパネル（ボタン付きEmbed）を設置する
  - パネルには「ガチャを引く」ボタンと「ギャンブルをする」ボタンの両方を配置する
  - パネルは常設・使い回し可能
- ガチャ・ギャンブルの結果は、別途設定した**結果通知チャンネル**に**全員が見える形**で送信する（ephemeralではない）
- ユーザーはパネルのボタンを押すことでガチャ券を1枚消費してガチャを実行する
- ガチャ結果（確率・賞品）:

  | 等級 | 確率 | 獲得P |
  |------|------|-------|
  | 1等 | 3% | 500P |
  | 2等 | 7% | 300P |
  | 3等 | 10% | 100P |
  | 4等 | 30% | 50P |
  | はずれ | 50% | 10P |

- `/status` : 自分のガチャ券枚数・現在のP・今週のVC滞在時間を表示する

### 4. VCランキングシステム

- VC滞在時間の**月間累計**ランキングを管理する
- `/ranking` : 今月のVC滞在時間ランキングをコマンドで表示する
- 月末（**23:59**）まで集計し、翌月頭（**00:01**）に自動リセット
- リセット直前に設定済みのテキストチャンネルへランキングを自動投稿する

### 5. ギャンブル機能

- ガチャと同じ統合パネル（§3）上の「ギャンブルをする」ボタンで実行する（専用パネルコマンドは不要）
- ユーザーはボタンを押すことで100Pを消費してギャンブルを実行する（掛け金は返還されない）
- ギャンブル結果（確率・獲得P）:

  | 結果 | 確率 | 獲得P |
  |------|------|-------|
  | 大当たり | 0.001% | 3000P |
  | 当たり① | 20% | 500P |
  | 当たり② | 10% | 800P |
  | 全ロス | 0.001% | 0P（全残高消失） |
  | はずれ | 44.998% | 0P |
  | 小当たり① | 15% | 5P |
  | 小当たり② | 10% | 10P |

  > 合計: 100%

- 「全ロス」が発生した場合、結果通知チャンネルに以下のロールすべてをメンションしてユーザーへの全ロスを通知する
  - `<@&1469319867294417051>`
  - `<@&1469319364300898305>`
  - `<@&1469335516036989140>`

### 6. ショップ機能

- `/shop` : ショップ一覧を表示する
- `/buy <商品ID>` : 商品を購入する
- 初期商品: ロール付与など（詳細は追加予定）
- 商品・価格は管理者コマンドで追加・削除・編集可能にする

### 7. 管理者コマンド

管理者権限（`Administrator`）保持者のみ実行可能。

| コマンド | 内容 |
|----------|------|
| `/admin setchannel ranking <#ch>` | 月間ランキング自動投稿チャンネルを設定 |
| `/admin setchannel result <#ch>` | ガチャ・ギャンブル結果通知チャンネルを設定 |
| `/admin setchannel log <#ch>` | Pの変動ログ送信チャンネルを設定 |
| `/admin gacha panel` | ガチャ＋ギャンブル統合パネルを現在のチャンネルに設置 |
| `/admin addexclude <カテゴリ>` | ポイントカウント対象外カテゴリを追加（カテゴリ選択式） |
| `/admin removeexclude <カテゴリ>` | 対象外カテゴリを解除（カテゴリ選択式） |
| `/admin listexclude` | 対象外カテゴリ一覧を表示 |
| `/admin give @ユーザー <金額>` | 指定ユーザーにPを付与 |
| `/admin take @ユーザー <金額>` | 指定ユーザーからPを徴収 |
| `/admin addshopitem <名前> <価格> <ロール>` | ショップ商品を追加（ロール選択式） |
| `/admin removeshopitem <商品ID>` | ショップ商品を削除 |
| `/admin resetranking` | 月間ランキングを手動リセット |
| `/admin resetusers` | usersテーブルの全データを初期値にリセット |

### 9. Pの変動ログ機能

- ユーザーのP残高が変動した際、管理者が設定したログチャンネルに自動で記録する
- 記録対象となるP変動の要因:
  - VC滞在によるポイント付与
  - ガチャ実行・報酬付与
  - ギャンブル実行・報酬付与（全ロス含む）
  - 送金（送金元・受取先の両方を記録）
  - 請求承認による送金
  - 管理者による付与・徴収
  - ショップ購入
- ログの形式: 日時・対象ユーザー・変動量・変動後残高・変動要因を含むEmbed
- ログチャンネル未設定時はログを送信しない（エラーにしない）

### 8. ユーザー向けその他コマンド

| コマンド | 内容 |
|----------|------|
| `/balance` | 自分の現在のP残高を表示 |
| `/status` | ガチャ券枚数・P残高・今週のVC滞在時間を表示 |
| `/ranking` | 今月のVCランキングを表示 |

> ガチャ・ギャンブルの実行はコマンドではなくパネルのボタン操作のみ（ユーザーが直接コマンドで引くことはできない）

---

## 非機能要件

- **セキュリティ**: トークン・設定値は `.env` で管理し、リポジトリには含めない
- **永続化**: SQLite によりBot再起動後もデータが保持される
- **冪等性**: VC状態管理はBot再起動時に不整合が起きないよう考慮する
- **スケーラビリティ**: 単一Discordサーバー運用を想定
- **ロギング**: エラーはコンソールに出力する

---

## スコープ外（やらないこと）

- 複数サーバー対応
- Web管理画面
- ショップの具体的な商品内容（追加予定・別途設計）
- ポイント売買・外部通貨連携
